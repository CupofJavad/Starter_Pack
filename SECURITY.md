# Security Policy

## Supported Versions

This repository is a starter/template package.
Only the default branch (usually `main`) is actively maintained.

If you are using a derived project created from this template,
security responsibility transfers to that project owner.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

DO NOT:
- Open a public GitHub issue
- Post the vulnerability details in discussions or comments
- Commit proof-of-concept exploits to the repository

INSTEAD:
1. Send a private report via one of the following:
   - GitHub Security Advisories (preferred, if enabled)
   - Direct message to the repository owner (GitHub profile)

2. Include the following information:
   - Description of the vulnerability
   - Steps to reproduce (minimal and clear)
   - Affected files, modules, or dependencies
   - Potential impact (data exposure, RCE, privilege escalation, etc.)
   - Suggested mitigation (if known)

3. If applicable, include:
   - Version numbers
   - OS / runtime details
   - Explanation with secrets redacted

---

## Response Expectations

- Acknowledgement within a reasonable time
- Triage of severity and scope
- Fix prioritized by impact
- Coordinated disclosure if necessary

---

## Scope Notes

This repository:
- Does NOT accept responsibility for downstream projects
- Does NOT include hosted services or production infrastructure
- Does NOT store secrets (credentials must be provided via environment variables)

---

## Safe Harbor

Good-faith security research conducted in alignment with this policy
will not result in legal action or retaliation.

If you are unsure whether your finding qualifies as a security issue,
err on the side of reporting it privately.
